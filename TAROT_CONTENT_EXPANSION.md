## 현재 카드 데이터 상태

현재 앱의 실제 랜덤 카드 풀은 MVP 기준 샘플 10장이다.

* 바보
* 마법사
* 여사제
* 여황제
* 황제
* 교황
* 연인
* 전차
* 힘
* 은둔자

선택 화면에 보이는 12장은 실제 카드 종류가 아니라 사용자가 고르는 카드 뒷면 슬롯 수다.

## 78장 이미지 확장 정리

전체 78장 이미지 제작 프롬프트는 아래 파일에 정리되어 있다.

* `CARD_IMAGE_PROMPTS_78.md`

앱 확장용 78장 이미지 매니페스트는 아래 파일에 정리되어 있다.

* `src/data/tarotImageManifest.ts`

구성:

* Major Arcana 22장
* Wands 14장
* Cups 14장
* Swords 14장
* Pentacles 14장

## 78장 실제 적용 순서

1. `CARD_IMAGE_PROMPTS_78.md` 기준으로 `1024x1536 PNG` 카드 이미지를 생성한다.
2. 생성한 이미지를 `assets/cards/`에 `imageKey.png` 파일명으로 넣는다.
3. `src/assets/images.ts`에 새 이미지 require 매핑을 추가한다.
4. `src/data/tarotCards.ts`의 카드 데이터 풀을 10장에서 22장 또는 78장으로 확장한다.
5. 카드별 해석 문장을 보강한다.
6. `npm test`, `npm run typecheck`, `npx expo export --platform web --output-dir dist`로 검증한다.

## 권장 확장 방식

바로 78장을 한 번에 앱 랜덤 풀에 넣기보다, 이미지와 해석 품질을 확인하면서 아래 순서로 확장하는 것이 안전하다.

1. 10장 유지 + 상세 해석 품질 개선
2. Major Arcana 22장 적용
3. Minor Arcana 56장 추가
4. 전체 78장 랜덤 풀 적용

이 순서를 따르면 이미지가 없는 카드가 결과에 나와 fallback UI로 보이는 문제를 피할 수 있다.
