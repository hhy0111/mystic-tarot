# Mystic Tarot Draw Method Plan

## 현재 적용 방식

78장 전체 덱을 무작위로 섞은 뒤 3개의 부채꼴 묶음으로 모두 펼친다.

사용자는 화면에 펼쳐진 78장 전체 카드 중 마음이 가는 카드 3장을 직접 고른다.

선택 전에는 `다시 섞기`로 78장 전체 덱을 다시 섞어 펼칠 수 있다. 한 장이라도 선택한 뒤에는 이번 리딩의 일관성을 위해 다시 섞기를 막는다.

선택 순서는 그대로 아래 의미에 연결된다.

1. 현재 흐름
2. 숨은 원인
3. 앞으로의 방향

## 왜 변경했는가

기존 MVP 방식은 화면에 12장 뒷면이 보이지만 실제 결과 카드는 내부에서 미리 3장만 뽑혀 있었다. 78장 전체 덱이 적용되면 사용자는 많은 카드 중에서 직접 고르는 느낌을 기대한다.

따라서 이제는 `78장 덱 -> 3개 부채꼴 전체 펼침 -> 사용자가 3장 선택` 구조가 맞다.

한 화면에 78장을 하나의 부채꼴로 펼치면 카드가 너무 겹쳐 실제 터치 영역이 좁아진다. MVP에서는 아래 3묶음으로 나눠 전체 덱 감성과 조작성을 함께 확보한다.

1. 메이저 아르카나 22장
2. 컵 · 완드 28장
3. 소드 · 펜타클 28장

## 성공 기준

* 78장 전체 카드가 중복 없이 펼쳐진다.
* 사용자는 78장 중 3장까지만 선택할 수 있다.
* 선택 전에는 전체 덱을 다시 섞어 펼칠 수 있다.
* 한 장 선택 후에는 후보 카드가 고정된다.
* 선택한 카드가 실제 결과 카드가 된다.
* 정방향/역방향은 후보 카드마다 랜덤으로 결정된다.
* 결과 화면의 카드 순서는 사용자가 고른 순서를 따른다.

## UX 원칙

* 모바일에서는 3개 부채꼴 묶음으로 전체 덱을 보여준다.
* 데스크톱 웹 미리보기에서는 앱형 화면 폭 안에서 부채꼴이 더 넓게 펼쳐지도록 한다.
* 3장 선택 완료 후 결과 버튼이 바로 보여야 한다.
* 선택 슬롯에는 카드 위치 의미를 함께 표시한다.
* 78장이라는 확장감을 문구로 알려 사용자가 카드 풀이 커졌다는 것을 느끼게 한다.

## 추후 확장안

### 빠른 3장 리딩

현재 기본 방식이다.

용도:

* 오늘의 운세
* 연애운
* 금전운
* 선택의 고민
* 로또 번호

### 한 장 리딩

매일 접속용 리추얼에 적합하다.

용도:

* 오늘의 카드
* 오늘의 조언
* 오늘 피할 행동

### 5장 리딩

광고 보상 또는 프리미엄 확장에 적합하다.

예시 위치:

1. 현재 상황
2. 방해 요소
3. 도와주는 힘
4. 사용자가 할 행동
5. 가까운 결과

### 주간 리딩

재방문 기능으로 좋다.

예시 위치:

* 월요일 흐름
* 중간 변수
* 주말 결과

## 현재 MVP에서 제외한 것

* 켈틱 크로스
* 10장 이상 리딩
* 카드 도감 연동
* 히스토리 기반 추천
* 유료 리딩

위 기능은 카드 78장 데이터와 히스토리 저장이 안정화된 뒤 확장하는 편이 좋다.
