# Mystic Tarot App Quality And Retention Plan

## 현재 앱 진단

Mystic Tarot은 현재 아래 흐름까지는 MVP로 잘 작동한다.

* 개인정보 입력 없음
* 로그인 없음
* 서버 없음
* 실제 AI API 없음
* 7개 운세 항목 선택
* 12장 카드 UI에서 3장 선택
* 결과 화면 진입 시 순차 카드 공개 연출
* 무료 결과 + 광고 후 상세 해석
* 내부 데이터 기반 문장 조합

하지만 현재 구조는 사용자가 한 번 결과를 보고 나면 다음 행동이 약하다. 즉, 앱의 핵심 문제는 기능 부족보다 **재방문 이유 부족**이다.

## 앱 컨셉 재정의

현재 컨셉:

* 개인정보 없이 바로 보는 타로 운세 앱

추천 컨셉:

* 매일 3분, 오늘의 마음을 정리하는 신비로운 타로 리추얼 앱

핵심 방향:

* 타로를 단순 운세 결과가 아니라 `오늘의 감정 정리`, `작은 행동 제안`, `내일 다시 확인할 이유`로 연결한다.
* 사용자가 이름, 생년월일, 로그인 없이도 "내 상황을 읽어 준다"는 느낌을 받게 한다.
* 결과를 맞히는 앱보다, 오늘 어떤 행동을 하면 좋을지 알려 주는 앱으로 만든다.

## 재방문을 만드는 핵심 루프

추천 루프:

1. 오늘의 운세 또는 고민 선택
2. 카드 3장 선택
3. 무료 결과 확인
4. 광고 후 상세 해석 확인
5. 오늘 해볼 행동 1개 선택
6. 다음 접속 때 "지난 리딩 돌아보기" 또는 "오늘의 변화 확인"

이 루프에서 중요한 것은 결과 화면 이후다. 사용자가 "그래서 오늘 뭘 해보지?"까지 이어져야 다음 날 다시 앱을 열 이유가 생긴다.

## 우선 개선 방향

### 1. 오늘의 리추얼

앱을 켰을 때 매일 하나의 작은 리추얼을 제공한다.

예시:

* 오늘의 한 장
* 오늘의 세 장
* 오늘의 조언
* 오늘 피해야 할 행동
* 오늘 행운의 색상

기대 효과:

* 사용자가 매일 확인할 이유가 생긴다.
* 복잡한 기능 없이 재방문 루프가 생긴다.

구현 난이도:

* 낮음

추천 구현:

* `AsyncStorage`로 오늘 날짜의 리딩을 로컬 저장
* 하루에 한 번 "오늘의 카드" 고정
* 다음 날 새 카드 생성
* 개인정보나 서버 없이 가능

### 2. 오늘 해볼 행동 선택

상세 결과에 이미 `오늘 해볼 행동`, `상황별 추천 행동`이 있다. 여기서 사용자가 하나를 선택하게 만든다.

예시:

* "오늘 이 조언을 실천해볼게요"
* 선택한 행동을 결과 화면 하단에 고정
* 다음 접속 시 "어제 선택한 행동" 표시

기대 효과:

* 결과가 소비형 콘텐츠에서 행동형 콘텐츠로 바뀐다.
* 사용자가 다음 날 다시 열 이유가 생긴다.

구현 난이도:

* 낮음

주의:

* 서버 저장 금지
* 로그인 금지
* 로컬 저장만 사용

### 3. 지난 리딩 저장

현재는 결과를 보고 나면 사라진다. 재방문 앱이 되려면 최소한 최근 결과를 다시 볼 수 있어야 한다.

추천 범위:

* 최근 7개 리딩만 로컬 저장
* 저장 항목: 날짜, 운세 카테고리, 카드 3장, 점수, 한 줄 조언
* 상세 전체 저장은 나중에 확장

기대 효과:

* 사용자가 어제와 오늘을 비교할 수 있다.
* 앱이 단발성 운세가 아니라 기록 앱처럼 느껴진다.

구현 난이도:

* 중간

### 4. 카드 도감은 나중에

카드 도감은 재방문에는 도움이 되지만, 지금 바로 만들면 범위가 커진다.

추천 순서:

1. 먼저 22장 메이저 아르카나 이미지와 해석 적용
2. 이후 카드 도감 화면 추가
3. 사용자가 결과에서 본 카드만 도감에서 밝게 표시

기대 효과:

* 수집 욕구가 생긴다.
* 사용자가 "아직 못 본 카드"를 보기 위해 다시 리딩한다.

구현 난이도:

* 중간 이상

### 5. 78장 전체 확장

현재 실제 카드 풀은 10장이다. 재방문성을 올리려면 반복감을 줄여야 하므로 카드 수 확장은 중요하다.

추천 순서:

1. 10장 유지 + 상세 해석 개선
2. 메이저 아르카나 22장 적용
3. 전체 78장 이미지 생성
4. 78장 데이터와 해석 적용

기대 효과:

* 같은 카드가 반복되는 느낌 감소
* 앱이 더 진짜 타로 앱처럼 느껴짐

구현 난이도:

* 높음

## 화면별 완성도 개선

### LobbyScreen

현재 역할:

* 운세 카테고리 선택

개선 아이디어:

* 상단에 "오늘의 카드가 기다리고 있어요" 같은 일일 메시지 표시
* 오늘 이미 리딩했다면 "오늘의 리딩 다시 보기" 버튼 표시
* 카테고리 버튼 아래에 최근 본 운세 1개 표시

우선순위:

* 높음

### CardSelectScreen

현재 역할:

* 12장 중 3장 선택

개선 아이디어:

* 카드 선택 전 짧은 집중 문구 표시
* 선택한 카드가 하단으로 내려가는 연출 강화
* 3장 선택 완료 후 바로 결과 버튼이 glow
* 진동 피드백 추가 가능

우선순위:

* 중간

### ResultScreen

현재 역할:

* 무료 결과, 광고 상세 결과, 카드 공개 연출

개선 아이디어:

* 결과 첫 화면의 핵심 문장 더 강하게
* "오늘 해볼 행동 1개 선택" 추가
* "이 리딩 저장" 또는 자동 저장
* "내일 다시 확인하기" 메시지 추가
* 결과 공유용 이미지 생성은 나중에

우선순위:

* 매우 높음

## 추천 재방문 기능 목록

### A. 오늘의 카드

매일 앱을 열면 한 장의 카드가 나온다.

장점:

* 가장 단순하고 강한 재방문 구조
* 무료 기능으로 적합
* 광고 없이도 앱 사용 이유가 생김

### B. 오늘의 행동 체크

리딩 후 하나의 행동을 선택하고 다음 날 확인한다.

예시:

* "짧은 메시지 보내기"
* "자동결제 하나 확인하기"
* "오늘 결정하지 않을 것 정하기"

장점:

* 운세가 현실 행동으로 연결됨
* 사용자가 결과를 더 신뢰하게 됨

### C. 7일 운세 흐름

최근 7일 점수와 카드 흐름을 보여 준다.

장점:

* 기록을 보러 다시 들어온다.
* 단순 운세 앱에서 개인 리추얼 앱으로 바뀐다.

주의:

* 서버 없이 로컬 저장만 사용

### D. 카드 수집

결과에서 본 카드를 도감에 밝게 표시한다.

장점:

* 수집 동기 생김
* 78장 확장과 잘 맞음

주의:

* 지금 당장 구현하면 범위가 커진다.
* 22장 적용 후 시작하는 것이 좋다.

### E. 시즌 리딩

월요일, 보름달, 월말 등 특정 시점에 다른 리딩을 제공한다.

예시:

* 월요일: 이번 주의 흐름
* 금요일: 주말 관계운
* 월말: 다음 달 준비 카드

장점:

* 새 콘텐츠처럼 느껴짐
* 서버 없이 날짜 기반으로 가능

### F. 광고 보상 강화

현재 광고 보상은 상세 해석 공개다. 여기에 작은 보상을 더 붙일 수 있다.

예시:

* 광고 후 "오늘의 추가 조언" 1개 공개
* 광고 후 "카드 연결 해석" 강화
* 광고 후 "내 상황별 행동 3개" 공개

주의:

* 무료 결과가 너무 빈약하면 앱이 불친절하게 느껴진다.
* 광고 보상은 "더 깊은 해석"이어야지, 기본 이해를 막으면 안 된다.

### G. 카드 추천 로또 번호

선택한 세 장의 카드 정보로 1~45 사이 추천 번호 6개와 보너스 번호 1개를 만든다.

핵심 원칙:

* `당첨 예측`이 아니라 `카드 기반 재미용 추천 번호`로 표현
* 선택 카드 ID, 위치, 정방향/역방향을 섞어 번호 생성
* 같은 카드 조합이면 같은 번호가 나오고, 카드가 바뀌면 번호도 바뀜
* 구매 권유, 확률 상승, 당첨 보장 표현 금지

장점:

* 매주 다시 열 이유가 생긴다.
* 카드 선택 결과가 운세 외의 재미 요소로 확장된다.
* 서버나 외부 API 없이 내부 로직으로 구현 가능하다.

주의:

* 결과 화면에 `재미용`, `당첨 보장 아님`, `구매 권유 아님` 문구를 명확히 표시한다.
* 금전운과 혼동되지 않도록 카테고리명은 `로또 번호` 또는 `행운 번호`처럼 별도 표시한다.

### H. 오늘의 추천 리딩

앱을 열었을 때 날짜 기반으로 오늘 어울리는 운세 항목을 하나 추천한다.

핵심 원칙:

* 서버, 로그인, 개인정보 없이 날짜만으로 계산
* 같은 날짜에는 같은 추천이 유지됨
* 사용자가 직접 다른 카테고리를 고를 수 있어야 함
* 추천은 강요가 아니라 선택 피로를 줄이는 보조 장치로 사용

장점:

* 로비 화면이 매일 조금씩 달라지는 느낌을 준다.
* 처음 들어온 사용자가 무엇을 눌러야 할지 바로 알 수 있다.
* 구현 난이도는 낮지만 재방문 체감 효과가 있다.

### I. 접근성 기반 UX 보강

화려한 연출이 있더라도 버튼과 진행 상태는 분명히 읽혀야 한다.

개선 항목:

* 선택된 운세 버튼에 `selected` 상태 제공
* 비활성 버튼에 `disabled` 상태 제공
* 카드 선택 진행 상태를 숫자와 시각적 점으로 함께 표시
* 카드 위치 의미를 `현재 흐름`, `숨은 원인`, `앞으로 방향`으로 노출
* 결과 화면의 핵심 정보는 광고 전에도 이해 가능하게 유지

## MVP 이후 추천 구현 순서

### Phase 1. 재방문 최소 루프

목표:

* 매일 앱을 다시 열 이유 만들기

구현:

* 오늘의 카드
* 오늘의 리딩 로컬 저장
* 오늘 해볼 행동 선택
* Lobby에서 오늘 리딩 다시 보기

성공 기준:

* 사용자가 앱을 닫아도 오늘 결과를 다시 볼 수 있음
* 다음 날 새 카드가 나옴

### Phase 2. 콘텐츠 반복감 줄이기

목표:

* 같은 카드가 반복된다는 느낌 줄이기

구현:

* 메이저 아르카나 22장 적용
* 카드별 상황 예시 보강
* 운세 카테고리별 문장 세트 확장

성공 기준:

* 같은 카테고리를 여러 번 봐도 문장이 덜 반복됨

### Phase 3. 기록과 비교

목표:

* 사용자가 과거 리딩을 다시 보게 만들기

구현:

* 최근 7개 리딩
* 날짜별 카드/점수/조언 표시
* 어제 선택한 행동 확인

성공 기준:

* 결과 화면 외에 다시 볼 화면이 생김

### Phase 4. 수집과 장기 사용

목표:

* 장기 재방문 동기 만들기

구현:

* 카드 도감
* 본 카드 표시
* 78장 전체 적용
* 시즌 리딩

성공 기준:

* 사용자가 아직 못 본 카드나 새 리딩을 보러 다시 들어옴

## 현재 바로 올리면 좋은 품질 개선

우선순위 높은 순서:

1. 오늘의 카드 / 오늘의 리딩 저장
2. 결과 화면에서 오늘 해볼 행동 1개 선택
3. Lobby에서 오늘 리딩 다시 보기
4. 최근 7개 리딩 로컬 히스토리
5. 카드 추천 로또 번호 주간 리추얼
6. 오늘의 추천 리딩 고도화
7. 메이저 아르카나 22장 확장
8. 카드 선택 화면 사운드/진동 피드백
9. 카드 도감
10. 전체 78장 적용

## 유지해야 할 원칙

* 로그인 추가 금지
* 이름, 생년월일 입력 금지
* 서버 저장 금지
* 실제 AI API 호출 금지
* 로컬 저장 중심
* 무료 결과는 충분히 이해 가능해야 함
* 광고 상세는 더 깊은 해석으로 느껴져야 함
* 기능보다 매일 다시 열 이유를 우선함

## 최종 방향

Mystic Tarot은 단순히 "운세를 맞히는 앱"보다, 매일 사용자가 자기 마음과 행동을 정리하는 앱으로 가는 편이 더 강하다.

핵심 문장:

* 오늘의 카드를 뽑고, 오늘 해볼 행동 하나를 정하고, 내일 다시 확인하는 앱

이 방향이면 서버, 로그인, AI 없이도 충분히 재방문 구조를 만들 수 있다.
